<h2>📁 Repository structure</h2>

<pre><code>inputs/
models/
outputs/
info_synthetic/
README.md
</code></pre>

<ul>
  <li><code>inputs/</code>: input instances in <code>.dat</code> format, including the synthetic instances <code>R1, R2, ..., R36</code> and the sushi instance.</li>
  <li><code>models/</code>: Xpress/Mosel files implementing the compact exact formulation and the heuristic approach presented in the article, together with the Python script <code>instance_generator.py</code> used to generate the synthetic instances.</li>
  <li><code>outputs/</code>: output files generated by the models, together with an Excel summary of the obtained solutions.</li>
  <li><code>info_synthetic/</code>: text files containing additional information on the synthetic instances, including the group structure, rankings, weights, matrices, and objective values.</li>
</ul>

<hr>

<h2>📄 Format of the <code>.dat</code> files</h2>

<p>The folder <code>inputs/</code> contains synthetic instances and the sushi instance, all provided in plain text format.</p>

<h3>Synthetic instances</h3>

<p>Each synthetic instance contains:</p>

<ul>
  <li><code>n</code>: number of items</li>
  <li><code>g</code>: number of groups used to generate the instance</li>
  <li><code>nswaps</code>: maximum Kendall distance used as noise in the generation process</li>
  <li><code>a</code>: an <code>n × n</code> matrix associated with the instance</li>
</ul>

<p>The general structure is:</p>

<pre><code>n: &lt;number_of_items&gt;
g: &lt;number_of_groups&gt;
nswaps: &lt;maximum_kendall_distance&gt;
a:
[
  &lt;row_1&gt;
  &lt;row_2&gt;
  ...
  &lt;row_n&gt;
]
</code></pre>

<p>In the preprocessing step, the matrix <code>c</code> is computed from <code>a</code> as <code>c_rs = a_rs / (a_rs + a_sr)</code>.</p>

<h3>Sushi instance</h3>

<p>The sushi instance follows the same format, but it does not include the lines corresponding to <code>g</code> and <code>nswaps</code>.</p>

<p>Its general structure is:</p>

<pre><code>n: &lt;number_of_items&gt;
a:
[
  &lt;row_1&gt;
  &lt;row_2&gt;
  ...
  &lt;row_n&gt;
]
</code></pre>

<p>As in the synthetic instances, the matrix <code>c</code> is computed in the preprocessing step from <code>a</code> as <code>c_rs = a_rs / (a_rs + a_sr)</code>.</p>

<hr>

<h2>📄 Format of the synthetic information files</h2>

<p>Each file in <code>info_synthetic/</code> is provided in plain text format and is organized by groups.</p>

<p>The general structure is:</p>

<pre><code>GROUP 1
Ranking
...
Weight
...
Matrix
...
Objective value: ...

GROUP 2
Ranking
...
Weight
...
Matrix
...
Objective value: ...

...

TOTAL OBJECTIVE VALUE: ...
</code></pre>

<p>For each group, the file contains:</p>

<ul>
  <li><code>Ranking</code>: ordering associated with the group</li>
  <li><code>Weight</code>: weight assigned to the group</li>
  <li><code>Matrix</code>: matrix associated with the group, with all entries multiplied by <code>1000</code></li>
  <li><code>Objective value</code>: objective value corresponding to that group</li>
</ul>

<p>The last line reports the aggregated objective value for the whole instance.</p>

<hr>

<h2>⚙️ Generating the synthetic instances</h2>

<p>
The script <code>models/instance_generator.py</code> implements the procedure used to generate the synthetic instances considered in the computational experiments.
</p>

<p>To run the generator from the root directory of the repository, use:</p>

<pre><code>python models/instance_generator.py
</code></pre>

<p>
The script generates the corresponding <code>.dat</code> instance files and the additional text files containing information about their underlying group structure, central rankings, weights, matrices, and objective values.
</p>

<hr>

<h2>▶️ Using the optimization models</h2>

<p>
The Python files in <code>models/</code> implement the formulations and algorithms described in the article.
To solve an instance:
</p>

<ol>
  <li>Select the corresponding Python model file from <code>models/</code>.</li>
  <li>Specify a <code>.dat</code> file from <code>inputs/</code> as the input instance.</li>
  <li>Run the Python script to obtain the corresponding solution and objective value.</li>
  <li>Inspect the generated files in <code>outputs/</code> and, when applicable, the additional information in <code>info_synthetic/</code>.</li>
</ol>


<p>The meaning and purpose of each formulation and algorithm are described in detail in the article.</p>
