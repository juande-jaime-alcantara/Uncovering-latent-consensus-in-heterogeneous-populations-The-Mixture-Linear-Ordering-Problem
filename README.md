<h1>Uncovering latent consensus in heterogeneous populations: The Mixture Linear Ordering Problem — Supporting Material</h1>

<p>This repository contains the supplementary material associated with the article:</p>

<p><strong><em>“Uncovering latent consensus in heterogeneous populations: The Mixture Linear Ordering Problem”</em></strong></p>

<hr>

<h2>📁 Repository structure</h2>

<pre><code>inputs/
models/
outputs/
info_synthetic/
README.md
</code></pre>

<ul>
  <li><code>inputs/</code>: input instances in <code>.dat</code> format. This folder includes the synthetic instances <code>R1, R2, ..., R36</code> and the sushi instance.</li>
  <li><code>models/</code>: optimization model files, including the compact exact model and the heuristic model.</li>
  <li><code>outputs/</code>: output files generated by the models, together with a summary of excellence for the obtained solutions.</li>
  <li><code>info_synthetic/</code>: text files containing the additional information associated with the synthetic instances, including group structure, rankings, weights, and scaled matrices.</li>
  <li><code>README.md</code>: description of the repository structure, file formats, and usage instructions.</li>
</ul>

<hr>

<h2>Resources</h2>

<h3><code>inputs/</code></h3>

<p>The folder <code>inputs/</code> contains all the problem instances used in the computational experiments, stored in <code>.dat</code> format.</p>

<ul>
  <li><strong>Synthetic instances:</strong> <code>R1, R2, ..., R36</code>.</li>
  <li><strong>Benchmark instance:</strong> sushi instance.</li>
</ul>

<p>These files provide the input data required to execute the models included in the repository.</p>

<h3><code>models/</code></h3>

<p>The folder <code>models/</code> contains the optimization models implemented for the Mixture Linear Ordering Problem.</p>

<ul>
  <li><strong>Compact exact model</strong>: exact mathematical formulation used to solve the problem optimally.</li>
  <li><strong>Heuristic model</strong>: heuristic approach designed to obtain high-quality solutions with reduced computational effort.</li>
</ul>

<h3><code>outputs/</code></h3>

<p>The folder <code>outputs/</code> contains the results generated by the models for the tested instances.</p>

<ul>
  <li>Output files associated with each instance/model execution.</li>
  <li>A <strong>summary of excellence</strong> reporting the quality of the obtained solutions.</li>
</ul>

<h3><code>info_synthetic/</code></h3>

<p>The folder <code>info_synthetic/</code> contains auxiliary text files describing the synthetic instances in detail.</p>

<p>Each file is organized by groups. For every group, the following information is provided:</p>

<ul>
  <li><code>GROUP k</code>: identifier of the group.</li>
  <li><code>Ranking</code>: ordering associated with that group.</li>
  <li><code>Weight</code>: weight assigned to the group.</li>
  <li><code>Matrix</code>: matrix associated with the group, with all entries multiplied by <code>1000</code>.</li>
  <li><code>Objective value</code>: objective value corresponding to that group.</li>
</ul>

<p>At the end of each file, the overall value is reported as:</p>

<pre><code>TOTAL OBJECTIVE VALUE: &lt;value&gt;
</code></pre>

<h4>General structure</h4>

<pre><code>GROUP 1
Ranking
...
Weight
...
Matrix
...
Objective value: ...

GROUP 2
Ranking
...
Weight
...
Matrix
...
Objective value: ...

...

TOTAL OBJECTIVE VALUE: ...
</code></pre>

<h4>Meaning of the fields</h4>

<ul>
  <li><code>GROUP k</code>: identifies the corresponding group in the synthetic instance.</li>
  <li><code>Ranking</code>: ordered list associated with the alternatives/items of that group.</li>
  <li><code>Weight</code>: relative importance assigned to that group.</li>
  <li><code>Matrix</code>: numerical matrix associated with the group. Its entries are stored after being multiplied by <code>1000</code>.</li>
  <li><code>Objective value</code>: contribution of that group to the objective value.</li>
  <li><code>TOTAL OBJECTIVE VALUE</code>: aggregated objective value over all groups.</li>
</ul>

<p><strong>Note:</strong> matrix entries are displayed scaled by a factor of <code>1000</code>. Therefore, original values can be recovered by dividing each entry by <code>1000</code>.</p>

<h2>▶️ Using the repository</h2>

<ol>
  <li>Select one of the <code>.dat</code> files from <code>inputs/</code>.</li>
  <li>Open the desired model from <code>models/</code> (compact exact model or heuristic model).</li>
  <li>Execute the model using the selected instance.</li>
  <li>Inspect the generated results in <code>outputs/</code>.</li>
  <li>Consult the corresponding file in <code>info_synthetic/</code> for additional information on synthetic groups, rankings, weights, and matrices.</li>
</ol>
